========================================
STOCK-QUANT-OOP PIPELINE WALKTHROUGH
========================================

OBJECTIF
--------
Construire une table finale:
    research_features_daily

Cette table est:
- alignée sur les prix (univers investissable)
- enrichie avec momentum, trend, volatility, short
- sans duplication
- sans leakage

----------------------------------------
PIPELINE GLOBAL
----------------------------------------

symbol sources (Nasdaq + SEC)
        ↓
symbol_reference_source_raw
        ↓
symbol_reference
        ↓
market_universe (common stock only)
        ↓
price_source_daily_raw_all (stooq)
        ↓
price_source_daily_raw (filtered universe)
        ↓
price_history (compat legacy)
        ↓
price_bars_unadjusted
        ↓
price_bars_adjusted
        ↓
instrument_master
        ↓
feature_price_momentum_daily
feature_price_trend_daily
feature_price_volatility_daily
feature_short_daily
        ↓
research_features_daily (FINAL TABLE)

IMPORTANT:
- momentum = table maître
- LEFT JOIN trend / volatility / short
- aucune duplication
- aucune inflation de lignes

VERIFICATION

python3 <<'PY'
import duckdb

    COUNT(*) AS rows,
    COUNT(DISTINCT symbol) AS symbols,
    MIN(as_of_date),

""").fetchone()

----------------------------------------
NOTES IMPORTANTES
----------------------------------------

1. Univers maître = PRIX
   → garantit cohérence backtest

2. Short = enrichissement
   → LEFT JOIN uniquement

3. Aucun FULL OUTER JOIN
   → sinon explosion du nombre de lignes

4. Pipeline SQL-first
   → reproductible
   → scalable
